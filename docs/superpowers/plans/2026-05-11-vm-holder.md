# VM Holder Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the 5min polling heartbeat with a persistently-running `wsl.exe ... sleep infinity` child process that pins the WSL VM in an alive state, so clautel.service stays online 24/7 and the Telegram bot is always responsive.

**Architecture:** A single new `DevEnvUbuntu-WSL-VMHolder` Scheduled Task (AtStartup + S4U) runs `wscript.exe vm-holder.vbs`. The VBS spawns a hidden `wsl.exe ... sleep infinity` child (tagged with a recognizable command-line signature), then enters a `Do While True` loop that every 5 minutes (a) checks via WMI whether that signature is still in any `wsl.exe` command line (respawns if not), (b) probes `systemctl --user is-active clautel.service` and writes one log line. The Scheduled Task's `RestartOnFailure` is the outer safety net.

**Tech Stack:** PowerShell 5.1 (Windows setup), VBScript (runtime keepalive logic), bash (Linux verify + linger), Windows Task Scheduler, WSL2.

## Task 0: Reconcile spec §3.3 with reality (`WshShell.Exec` cannot hide windows)

**Why this task exists:** The vm-holder spec §3.3 sketches `WshShell.Exec` for both holding sleep infinity and probing clautel. But the prior `keepalive-hide-probe-window` spec already established that `WshShell.Exec` forces `SW_SHOWNORMAL` and cannot hide its child window — that's exactly why the current v2 heartbeat uses `WshShell.Run cmd, 0, True` with cmd.exe redirect. Carrying Exec into the holder design would reintroduce flashing black windows.

The fix: spawn sleep infinity with `WshShell.Run holdCmd, 0, False` (vbHide, fire-and-forget); detect liveness by querying `Win32_Process` for any `wsl.exe` whose `CommandLine` contains a hard-coded signature substring; spawn again if none found. Probe clautel exactly the way current `wsl-heartbeat.vbs` does (Run+temp file). No `WshShell.Exec` anywhere.

```powershell

  # v3: 不再轮询心跳, 改用持续持有 wsl.exe sleep infinity 子进程模式.
  # 见 docs/superpowers/specs/2026-05-11-vm-holder-design.md.

  $logPath  = Join-Path $launcherDir "holder.log"

  $signature = "DevEnvUbuntu-vm-holder:${Distro}:${WslUser}"

  # PowerShell 字符串里 ` 是 escape, " 用 "" 转义; 用 here-string 简化
  $vbs = @"
' DevEnvUbuntu v3 vm-holder.vbs
' Auto-generated by setup-keepalive.ps1 - do not edit by hand
Option Explicit

Const DISTRO       = "$Distro"
Const WSL_USER     = "$WslUser"
Const SIGNATURE    = "$signature"
Const HOLDER_SIG   = "$holderSig"
Const LOG_PATH     = "$($logPath  -replace '\\','\\')"
Const PROBE_TMP    = "$($probeTmp -replace '\\','\\')"
Const MAX_LOG_BYTES = 1048576    ' 1 MB rotate threshold
Const LOOP_INTERVAL_MS = 300000  ' 5 min between probes
Const RESPAWN_GRACE_MS = 3000    ' wait 3s after spawning sleep infinity

' --- helper: append a timestamped log line, rotate at 1MB --------------------
Sub LogLine(level, msg)
  Dim fso, f
  Set fso = CreateObject("Scripting.FileSystemObject")
  If fso.FileExists(LOG_PATH) Then
    If fso.GetFile(LOG_PATH).Size > MAX_LOG_BYTES Then
      Dim rotated : rotated = LOG_PATH & ".1"
      If fso.FileExists(rotated) Then fso.DeleteFile rotated, True
      fso.MoveFile LOG_PATH, rotated
    End If
  End If
  Set f = fso.OpenTextFile(LOG_PATH, 8, True)
  Dim ts : ts = FormatDateTime(Now, 0)
  f.WriteLine ts & " [" & level & "] " & msg
  f.Close
End Sub

' --- helper: query WMI for live wsl.exe with our signature in CommandLine ----
Function IsHolderAlive()
  Dim wmi, procs, p
  IsHolderAlive = False
  On Error Resume Next
  Set wmi = GetObject("winmgmts:\\.\root\cimv2")
  If Err.Number <> 0 Then Exit Function
  Set procs = wmi.ExecQuery( _
    "SELECT ProcessId, CommandLine FROM Win32_Process WHERE Name = 'wsl.exe'")
  For Each p In procs
    If Not IsNull(p.CommandLine) Then
      If InStr(p.CommandLine, HOLDER_SIG) > 0 Then
        IsHolderAlive = True
        Exit Function
      End If
    End If
  Next
  On Error GoTo 0
End Function

' --- helper: read probe.tmp into a string, delete file, strip CR/LF ---------
Function ReadAndDeleteProbeTemp()
  Dim fso, f, s
  Set fso = CreateObject("Scripting.FileSystemObject")
  ReadAndDeleteProbeTemp = ""
  If fso.FileExists(PROBE_TMP) Then
    Set f = fso.OpenTextFile(PROBE_TMP, 1)
    If Not f.AtEndOfStream Then s = f.ReadAll() Else s = ""
    f.Close
    fso.DeleteFile PROBE_TMP, True
    s = Replace(s, vbCrLf, "")
    s = Replace(s, vbLf,   "")
    s = Replace(s, vbCr,   "")
    ReadAndDeleteProbeTemp = Trim(s)
  End If
End Function

' --- (a) wscript exclusion: refuse to run if another vm-holder.vbs is up ----
Dim wmi, procs, p, dupCount
On Error Resume Next
Set wmi = GetObject("winmgmts:\\.\root\cimv2")
Set procs = wmi.ExecQuery( _
  "SELECT ProcessId, CommandLine FROM Win32_Process WHERE Name = 'wscript.exe'")
dupCount = 0
For Each p In procs
  If Not IsNull(p.CommandLine) Then
    If InStr(p.CommandLine, "vm-holder.vbs") > 0 Then
      dupCount = dupCount + 1
    End If
  End If
Next
On Error GoTo 0
If dupCount > 1 Then
  WScript.Quit 0
End If

LogLine "INFO", "vm-holder started, pid=" & GetCurrentPid()

' --- (b) main loop: hold + probe every 5 minutes -----------------------------
Dim sh : Set sh = CreateObject("WScript.Shell")
Dim holdCmd, coreCmd, probeCmd, exitCode, statusOutput

holdCmd = "wsl.exe -d " & DISTRO & " -u " & WSL_USER & _
          " -e /bin/bash -c ""exec /bin/sleep infinity # " & HOLDER_SIG & """"
coreCmd = "wsl.exe -d " & DISTRO & " -u " & WSL_USER & _
          " -e systemctl --user is-active clautel.service"
probeCmd = "cmd.exe /c """ & coreCmd & " > """ & PROBE_TMP & """ 2>&1"""

Do While True
  ' (b1) hold: respawn if dead
  If Not IsHolderAlive() Then
    LogLine "WARN", "holder child not found, respawning"
    sh.Run holdCmd, 0, False   ' 0=vbHide, False=fire-and-forget
    LogLine "INFO", "vm holder attached (sleep infinity spawned)"
    WScript.Sleep RESPAWN_GRACE_MS
  End If

  ' (b2) probe clautel.service
  exitCode = sh.Run(probeCmd, 0, True)
  statusOutput = ReadAndDeleteProbeTemp()

  ' (b3) three-state log
  If exitCode = 0 And statusOutput = "active" Then
    LogLine "OK", "clautel.service active"
  ElseIf statusOutput = "activating" Then
    LogLine "INFO", "clautel.service activating, give it time"
  Else
    LogLine "WARN", "clautel down (status='" & statusOutput & _
                     "', exit=" & exitCode & "), bootstrapping  # " & SIGNATURE
    Dim bootCmd
    bootCmd = "wsl.exe -d " & DISTRO & " -u " & WSL_USER & _
              " -e systemctl --user start clautel.service"
    sh.Run bootCmd, 0, False
    LogLine "INFO", "boot trigger fired"
  End If

  ' (b4) sleep 5 min until next iteration
  WScript.Sleep LOOP_INTERVAL_MS
Loop

' --- helper: VBScript has no WScript.PID; shell out for our own PID ---------
Function GetCurrentPid()
  Dim shp, fso2, pidFile, f, sPid
  Set shp = CreateObject("WScript.Shell")
  pidFile = shp.ExpandEnvironmentStrings("%LOCALAPPDATA%") & "\DevEnvUbuntu\holder.pid.tmp"
  shp.Run "cmd.exe /c echo %errorlevel% > """ & pidFile & """", 0, True
  ' fallback: query WMI for self
  Dim wmi2, procs2, p2, myPpid
  GetCurrentPid = 0
  On Error Resume Next
  Set wmi2 = GetObject("winmgmts:\\.\root\cimv2")
  Set procs2 = wmi2.ExecQuery( _
    "SELECT ProcessId, ParentProcessId, CommandLine FROM Win32_Process WHERE Name = 'wscript.exe'")
  For Each p2 In procs2
    If Not IsNull(p2.CommandLine) Then
      If InStr(p2.CommandLine, "vm-holder.vbs") > 0 Then
        GetCurrentPid = p2.ProcessId
        Exit Function
      End If
    End If
  Next
  On Error GoTo 0
End Function
"@

  # write the file (ASCII; no BOM; wscript handles it)
  $vbs | Set-Content -Path $vbsPath -Encoding Ascii -Force
```

`new_string` =:

```powershell
  # === 6) 注册任务: 单 trigger, RestartOnFailure 外层兜底 ====================
  $taskName = "DevEnvUbuntu-WSL-VMHolder"
  $wscript  = Join-Path $env:SystemRoot "System32\wscript.exe"

  $action = New-ScheduledTaskAction -Execute $wscript `
    -Argument ('"{0}"' -f $vbsPath)

  $triggerStartup = New-ScheduledTaskTrigger -AtStartup

  # v3 没有 5min 心跳 trigger - VBS 自己永不退出
  $settings = New-ScheduledTaskSettingsSet `
    -AllowStartIfOnBatteries -DontStopIfGoingOnBatteries `
    -StartWhenAvailable `
    -ExecutionTimeLimit ([TimeSpan]::Zero) -Hidden `
    -MultipleInstances IgnoreNew `
    -RestartCount 999 `
    -RestartInterval (New-TimeSpan -Seconds 30)

  $principal = New-ScheduledTaskPrincipal `
    -UserId "$env:USERDOMAIN\$env:USERNAME" -LogonType S4U -RunLevel Highest

  Unregister-ScheduledTask -TaskName $taskName -Confirm:$false -ErrorAction SilentlyContinue
  Register-ScheduledTask -TaskName $taskName `
    -Action $action `
    -Trigger @($triggerStartup) `
    -Settings $settings -Principal $principal | Out-Null
  Write-Host "[OK] 已注册任务计划: $taskName (AtStartup, S4U, RestartOnFailure=999x30s)"

  # === 6b) 自检注册结果 (防止后续被人意外改回 AtLogOn 或丢掉 S4U) ============
  $registered = Get-ScheduledTask -TaskName $taskName
  $bootTrigger = $registered.Triggers | Where-Object {
    $_.CimClass.CimClassName -eq 'MSFT_TaskBootTrigger'
  }
  if (-not $bootTrigger) {
    throw "[ASSERT] 任务注册成功了, 但没找到 AtStartup (MSFT_TaskBootTrigger). 检查 setup-keepalive.ps1 中 trigger 写法."
  }
  if ($registered.Principal.LogonType -ne 'S4U') {
    throw "[ASSERT] 任务 principal LogonType = '$($registered.Principal.LogonType)', 期望 S4U. 检查 New-ScheduledTaskPrincipal 参数."
  }
  if ($registered.Settings.RestartCount -lt 1) {
    throw "[ASSERT] 任务 RestartCount = '$($registered.Settings.RestartCount)', 期望 >=1. 检查 New-ScheduledTaskSettingsSet -RestartCount."
  }
  Write-Host "[OK] 配置断言通过: AtStartup trigger + S4U principal + RestartOnFailure"
```
